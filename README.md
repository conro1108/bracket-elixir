# Bracket

A real-time tournament bracket app built with Elixir + Phoenix LiveView. Create a bracket with any items, share a link, and let participants vote head-to-head until a champion emerges. No accounts required.

Rewrite of [bracket](../bracket/) (Node.js/Socket.io) as an Elixir learning project.

## Features

- **Create** a bracket with 4–32 items and a host display name
- **Share** a join link — participants join with a display name, no login needed
- **Vote** in real-time; vote counts update live for all participants
- **Auto-close** matchups when all eligible participants have voted
- **Timer** (optional, per matchup) with configurable countdown
- **Host controls**: close matchup early, kick participants, restart bracket
- **Host recovery** link for reconnecting after a page refresh
- **Automatic host transfer** if the host disconnects for too long
- **Inactivity cleanup** — brackets self-destruct after 4 hours of inactivity

## Architecture

```
lib/bracket/
  game.ex           # Pure functions: seeding, voting, closing, bracket state
  bracket_server.ex # GenServer per bracket (Registry + DynamicSupervisor)
  sanitizer.ex      # Input trimming/truncation

lib/bracket_web/
  live/home_live.ex     # Bracket creation form
  live/bracket_live.ex  # Single LiveView: join, lobby, voting, finished
  components/
    bracket_tree.ex     # Bracket visualization component
  controllers/
    session_controller.ex  # PUT session workaround for LiveView
    health_controller.ex   # GET /health for load balancer checks
```

### Key design decisions

**One GenServer per bracket.** State lives in memory in the BEAM process. No database. The process dies with the bracket (inactivity timeout or server restart). Supervised under `Bracket.DynamicSupervisor` with `restart: :transient` and `max_children: 1000`.

**Pure game logic.** `Bracket.Game` contains only pure functions — no PubSub, no Process calls. `BracketServer` orchestrates side effects (broadcasting, timers, monitoring). This makes the core logic fully unit-testable.

**Session management via redirect.** Phoenix LiveView `handle_event` cannot call `put_session`. On bracket creation and participant join, the LiveView signs credentials into a short-lived `Phoenix.Token` (30s TTL) and redirects through `SessionController`, which verifies the token, writes the signed session cookie, and redirects back. Credentials are never in URLs.

**Vote anonymity.** `Game.public_view/1` strips individual vote maps from all PubSub broadcasts and get_state responses — replacing `%{participant_id => :a | :b}` with `%{count_a: n, count_b: n, total_eligible: n}`.

**Power-of-2 seeding.** Items are padded to the next power of 2. Byes are distributed to the top N items (first-in-order), not bunched at the end, so there are no (nil, nil) matchups and bye winners always advance cleanly.

## Setup

**Prerequisites:** Elixir 1.19+, OTP 28+

```bash
# Install dependencies
mix setup

# Start the dev server (hot reload enabled)
mix phx.server
# or in IEx
iex -S mix phx.server
```

Visit [http://localhost:4000](http://localhost:4000).

## Testing

```bash
# Run all tests except slow cleanup tests
mix test --exclude slow

# Run all tests including the inactivity cleanup test (~1s)
mix test
```

The test suite is 88 tests across:
- `test/bracket/game_test.exs` — pure Game logic (async: true)
- `test/bracket/bracket_server_test.exs` — GenServer integration (async: false)
- `test/bracket_web/live/` — LiveView feature tests (async: false)

Test timing is configurable in `config/test.exs` (cleanup intervals reduced to milliseconds).

## Environment variables

| Variable | Required | Description |
|---|---|---|
| `SECRET_KEY_BASE` | Production | 64-byte secret for signing sessions/tokens. Generate with `mix phx.gen.secret`. |
| `PHX_HOST` | Production | Public hostname (e.g. `bracket.example.com`) |
| `PORT` | Optional | HTTP port (default: 4000) |

## Deployment

A `Dockerfile` is included (generated by `mix phx.gen.release --docker`).

```bash
# Build release
docker build -t bracket .

# Run
docker run -p 4000:4000 \
  -e SECRET_KEY_BASE=$(mix phx.gen.secret) \
  -e PHX_HOST=your-host.example.com \
  bracket
```

For Render, Fly.io, or similar platforms, set `SECRET_KEY_BASE` and `PHX_HOST` as environment variables and deploy from the Dockerfile.

**Note:** All bracket state is in-memory. A container restart loses active brackets. This is intentional for v1.

## Known limitations

- No persistence — brackets are lost on restart
- No auth — anyone with the join URL can join
- Max 1000 concurrent brackets per node (configurable via `DynamicSupervisor` `max_children`)
- Max 50 participants per bracket
- Max 32 items per bracket
